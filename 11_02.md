## Задание 1. Кеширование

Проблемы, которые решает кеширование:

Кеширование – это хранение часто запрашиваемых данных в быстром промежуточном хранилище для ускорения доступа и снижения нагрузки на основные источники. Основные проблемы, решаемые кешированием:

1. Высокая задержка (latency)
Запрос к медленной дисковой базе данных или удалённому API может занимать десятки–сотни миллисекунд. Кеш в оперативной памяти (Memcached, Redis) отдаёт результат за микросекунды, резко сокращая время отклика.

2. Перегрузка баз данных и бэкенд-систем
При пиковых нагрузках (лавина запросов на одну и ту же страницу или данные) сервер БД может не справиться. Кеш берёт на себя большую часть повторяющихся запросов, не допуская перегрузки.

3. Вычислительная дороговизна формирования ответа
Генерация сложных отчётов, агрегаций, рендеринг страниц – ресурсоёмкие операции. Сохранённый в кеше результат избавляет от повторных вычислений, экономя CPU и время.

4. Снижение пропускной способности сети и задержек при географической распределённости
Кеширование на уровне CDN или прокси-серверов (например, статических ресурсов) доставляет контент с ближайшего к пользователю узла, уменьшая время передачи данных.

5. Недоступность основного источника данных (отказоустойчивость)
Если база данных временно недоступна, кеш может продолжать отдавать актуальные данные (в пределах TTL), обеспечивая частичную работоспособность сервиса.

6. Ограничения по числу операций (rate limiting)
Внешние API часто квотируют количество запросов. Кеширование ответов позволяет делать запрос реже, укладываясь в лимиты.

## Задание 3. Удаление по TTL в Memcached

Для работы с memcached понадобится утилита nc (netcat). Проверим её наличие у себя (если нет - установим):

```
sudo dnf install nmap-ncat -y   # содержит команду nc
```

<img width="852" height="245" alt="Screenshot from 2026-04-27 21-40-47" src="https://github.com/user-attachments/assets/57c23232-000a-4cbb-a419-0983b47917d1" />

Далее, для выполнения задания выполним в терминале заранее заготовленные команды (я привожу их с пояснениями):

```
# 1. Запись первого ключа
printf "set key1 0 5 6\r\nvalue1\r\n" | nc localhost 11211

# 2. Запись второго ключа
printf "set key2 0 5 6\r\nvalue2\r\n" | nc localhost 11211

# 3. Сразу получаем оба ключа (должны вернуться значения)
printf "get key1\r\nget key2\r\n" | nc localhost 11211

# 4. Ждём 6 секунд, чтобы TTL истёк
sleep 6

# 5. Повторно запрашиваем ключи (должны отсутствовать)
printf "get key1\r\nget key2\r\n" | nc localhost 11211
```
